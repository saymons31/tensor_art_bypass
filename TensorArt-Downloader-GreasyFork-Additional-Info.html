TensorArt Downloader adiciona um botão de download em gerações do Tensor.Art quando a interface marca o conteúdo como restrito/bloqueado e desativa o download nativo.

Baseado no userscript original de Angry Toenail (v0.2), com melhorias de robustez: contorno de CORS, várias fontes de URL, download real do arquivo e diagnóstico automático quando o site muda e o script quebra.

Recursos

  • Injeta botão Download em itens bloqueados na lista de tasks/gerações
  • Resolve a mídia por várias fontes: JSON da task, DOM, índice de respostas da API, e endpoint de download
  • Salva o arquivo de verdade (blob), em vez de só abrir em nova aba
  • Autenticação via sessão logada (cookie / Bearer)
  • Quase silencioso quando funciona; gera relatório só se falhar por possível atualização do site
  • API de debug no console: __TADL.help()

Requisitos

  • Navegador com Tampermonkey, Violentmonkey ou similar
  • Conta logada em https://tensor.art/
  • Permissões do script: GM_xmlhttpRequest e acesso a api.tensor.art (aceite quando o manager pedir)

Instalação

  1. Instale um gerenciador de userscripts (Tampermonkey / Violentmonkey).
  2. Clique em Install this script nesta página do Greasy Fork.
  3. Confirme a instalação e as permissões de domínio, se solicitadas.
  4. Abra tensor.art e faça login.
  5. Recarregue a página com Ctrl+F5.

Importante: desative outras cópias antigas do mesmo script. Duas versões ativas no mesmo site podem conflitar.

Como usar

  1. Entre na área de gerações / histórico de tasks onde há mídia com download bloqueado na UI.
  2. Aguarde o card carregar. O botão bloqueado deve virar Download.
  3. Clique em Download.
    • Done — arquivo salvo
    • Fail — falhou (veja diagnóstico abaixo)

No console do navegador (F12 → Console), ao carregar a página você deve ver algo como:

 [TA-DL] v0.3.7 ready | token=yes | GM_xhr=yes | ...
  • token=yes — cookie de sessão encontrado (logado)
  • GM_xhr=yes — permissão de rede do userscript OK

Como funciona (resumo técnico)

  1. Intercepta respostas da API que contêm /query (lista de tasks).
  2. Localiza no DOM os itens com classe de bloqueio (.group.cursor-not-allowed).
  3. Associa cada card ao item da API (ex.: imageId).
  4. Monta candidatos de URL, nesta lógica geral:
    1. URLs já vistas em respostas da API (índice em memória)
    2. Campos do JSON do item da task
    3. Tags <img> / <video> no card
    4. Recursos de rede da página que contêm o id
    5. Endpoint POST /works/v1/generation/image/download (fallback)
  5. Baixa o melhor candidato via GM_xmlhttpRequest (evita bloqueio CORS do fetch da página).

Diagnóstico automático (quebra por atualização do site)

Quando o Tensor.Art muda layout ou API e o script falha de forma “estrutural”, ele pode gerar sozinho um relatório:

  • Toast laranja na página
  • Grupo no console: [TA-DL] UPDATE DIAGNOSTIC
  • Download automático de ta-dl-diag-<timestamp>.json
  • Histórico em localStorage (últimos relatórios)

Em downloads bem-sucedidos o script permanece quase silencioso. Relatórios iguais são limitados (debounce ~90s) para não spammar.

O JSON do relatório inclui códigos de erro, o que provavelmente mudou, o que editar no script, probe de seletores DOM, amostra do JSON do item e tentativas da API.

Códigos mais comuns

Código Significado / onde olhar
PAYLOAD_NO_TASKS Resposta do /query sem data.tasks[]
DOM_TASK_NOT_FOUND Card da task não encontrado (CSS/layout mudou)
ITEM_MISSING_ID Item sem imageId (ou campos equivalentes)
ITEM_NO_URLS JSON do item sem URL de mídia
DOWNLOAD_API_FAIL Endpoint de download / auth / body falhou
NO_CANDIDATES Nenhuma URL utilizável encontrada
ALL_CANDIDATES_FAILED Havia URLs, mas todas falharam ou eram placeholder
TOKEN_MISSING Não logado / cookie de sessão ausente
GM_XHR_MISSING @grant GM_xmlhttpRequest não aplicado

Comandos no console

Com a página do Tensor.Art aberta:

__TADL.help()           // guia resumido
__TADL.diag.last()      // último relatório de quebra
__TADL.diag.history()   // histórico salvo
__TADL.diag.export()    // baixar de novo o último .json
__TADL.diag.clear()     // limpar histórico
__TADL.dump()           // índices internos + último diag
__TADL.contract         // seletores/paths que o script espera
__TADL.version          // versão do script

Solução de problemas

  • Botão não aparece
    • Confirme que está logado e que a lista de tasks recarregou
    • Veja se o console mostra token=yes e GM_xhr=yes
    • Desative outras cópias do script
    • O layout do site pode ter mudado → aguarde um relatório ou use __TADL.dump()
  • Fail ao clicar
    • Se baixou ta-dl-diag-*.json, abra e leia codes e whatToFix
    • Às vezes a API devolve só imagem “forbidden”/placeholder de moderação; o script tenta outras fontes no JSON/DOM
  • token=NO — faça login de novo e recarregue a página
  • GM_xhr=NO — reinstale o script e aceite as permissões do manager

Privacidade e segurança

  • O script roda só em tensor.art (conforme @match).
  • Usa o token de sessão já presente no navegador para chamar a API do próprio Tensor.Art.
  • Não envia seus dados para servidores de terceiros.
  • Relatórios de diagnóstico ficam no seu PC (download local + localStorage do site).
  • Como todo userscript, revise o código antes de instalar.

Limitações

  • Depende do layout e das APIs atuais do Tensor.Art; mudanças no site podem quebrar seletores ou endpoints.
  • Se o backend deixar de expor a mídia no JSON e só servir placeholder na API de download, pode ser impossível recuperar o arquivo só pelo browser.
  • A assinatura HMAC da API de download usa o esquema compatível com o script original (0.2). Se a API passar a validar sign de forma rígida, essa parte precisará de atualização.
  • Uso sob sua responsabilidade e de acordo com os termos do Tensor.Art e a legislação local.

Feedback

Se o script parar após uma atualização do Tensor.Art:

  1. Reproduza o erro uma vez
  2. Anexe o arquivo ta-dl-diag-*.json (se tiver sido gerado) no feedback/issue
  3. Ou cole a saída de __TADL.diag.last() e __TADL.help()

Isso indica exatamente qual parte mudou (DOM, payload ou API) e acelera a correção.
